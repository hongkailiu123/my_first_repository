Question 1.6
{points: 3}

Below you can see multiple initializations of k-means with different seeds for K = 4. Can you explain what is happening and how we can mitigate this in the kmeans function?

Student's answer Score: 3.0 / 3.0 (Top)

It seems that the K-means algorithm is getting “stuck” in a few bad solutions, and we cannot distinguish which one is the best clustering from the plots shown above. To solve this problem, we can try a few different random initializations via the nstart argument in the kmeans function. For example, if we write kmeans(scaled_km_data, nstart = 10, centers = 4), then K-means clustering will be performed 10 times, and the best clustering from this will be returned. Moreover, the more times K-means clustering is performed, the more likely we are to find the best clustering if the best one exists, but we need to consider the computation cost when choosing how many times K-means clustering will be performed.
